# Code Quality

- Read and understand files before editing; never modify code you haven't read
- Follow existing patterns and conventions in the codebase (indentation, naming, structure)
- Don't add dependencies without asking the user first
- Prefer editing existing files over creating new ones
- Don't over-engineer: no speculative abstractions, no unnecessary wrappers, no premature generalization
- Don't add comments, docstrings, or type annotations to code you didn't change
- Remove dead code completely rather than commenting it out
- Don't make changes beyond what was requested (no drive-by refactors)
